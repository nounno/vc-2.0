# VC 2.0 项目宪法 v1.0

**生效日期**: 2026-05-19

---

## 第一章：战略资产定义

九个战略资产，九个独立Docker容器：

| 战略资产 | 架构单元 | 商业计划书依据 |
|----------|----------|----------------|
| 五层数据工厂 | pipeline | 4.2节 |
| IR统一格式 | pipeline内实现 | 4.2节 |
| 数据中心 | datacenter | 4.4节 |
| 五大核心视图 | admin | 4.3节 |
| 供应商门户 | supplier | - |
| 品牌官网 | web | - |
| AI采购助手 | search | 5.2节 |
| 业务逻辑层 | api | - |
| 学习闭环 | pipeline内实现 | 4.2节 |

**强制要求**：
- 资产之间通过API接口通信，禁止共享数据库直连
- 新增战略资产必须同时修订本宪法

## 第五章：五大核心视图

设计哲学：从"人命令系统执行"向"系统主动引导人决策"转变。

## 第六章：AI采购助手

独立容器search，分三阶段实现：关键词匹配 → 结构化查询 → 自然语言理解

---

## 第七章：学习闭环

三次纠正稳定一条规则，写入Rule表

---

## 第八章：技术选型

- 前端：Next.js 14 + Shadcn UI
- 后端：TypeScript Next.js API Routes
- 数据库：MySQL 8.4
- 消息队列：Redis Streams（接入层异步化必装，研发环境搭建阶段即须部署）
- 缓存：Redis（搜索与数据中心预计算必装，研发环境搭建阶段即须部署）
- 包管理：Turborepo Monorepo
- AI Agent：九月（Hermes）
- 部署：Docker全容器化 + docker-compose

---

## 第九章：安全原则

- 最小权限
- 认证分离（内部API Key，外部JWT）
- 数据脱敏（供应商身份不可见）
- 审计可追溯

---

## 第十章：宪法修订程序

1. 修订提案由董事长提出
2. 修订草案由九月起草，董事长确认
3. 新版本写入docs/constitution.md，旧版本归档

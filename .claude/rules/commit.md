# Git 提交规范

## 提交信息格式
```
<type>: <subject>

<body>
```

## Type 类型
| Type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复 bug |
| `docs` | 文档更新 |
| `style` | 格式（不影响功能） |
| `refactor` | 重构（不影响功能） |
| `test` | 测试相关 |
| `chore` | 杂项（构建、依赖、工具） |
| `perf` | 性能优化 |

## 规则

1. **subject** 最多 50 字，描述做什么
2. **body** 可选，描述为什么改、以及与之前行为的差异
3. **禁止提交**：
   - `credentials.md`、`.env`、`.env.secrets`
   - `node_modules/`、`.next/`、`dist/`
   - 任何包含真实密钥/密码的内容

## 回滚规范

- 回滚原因必须说明
- 回滚后需要报告当前版本
- 回滚前必须确认目标版本

## 示例

```
feat: 添加供应商数据导出功能

支持按时间范围导出供应商报价数据，
解决了手动整理数据耗时的问题。
```

```
fix: 修复 pipeline 数据清洗丢失小数问题

之前会截断小数位，现在保留2位小数。
```
